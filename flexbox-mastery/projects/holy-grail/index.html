Project: Holy Grail Layout

Build the classic three-column layout with flexbox

üéØ What is the Holy Grail Layout?

The "Holy Grail" layout is a classic web design pattern with:

  • A fixed-height header
  • A fixed-height footer
  • A fluid center with:
    • A fixed-width navigation on the left
    • A fixed-width sidebar on the right
    • A fluid-width main content area in the middle

This layout was notoriously difficult to implement before flexbox, often requiring complex float-based solutions or absolute positioning hacks.

This is the main content area of the Holy Grail layout. It takes up the available space between the two sidebars.

The beauty of flexbox is that this center column automatically expands to fill the available space, while the sidebars maintain their specified widths.

Before flexbox, this layout required complex CSS with floats, negative margins, and other hacks.

Now it's simple and straightforward with just a few flexbox properties!

üíª Implementation

<div class="holy-grail-container">
    <header class="hg-header">Header</header>
    <div class="hg-body">
        <nav class="hg-nav">Navigation</nav>
        <main class="hg-content">Main Content</main>
        <aside class="hg-sidebar">Sidebar</aside>
    </div>
    <footer class="hg-footer">Footer</footer>
</div>
/* Container with flex column */
.holy-grail-container {
    display: flex;
    flex-direction: column;
    min-height: 100vh; /* Full viewport height */
}

/* Header and footer with fixed height */
.hg-header, .hg-footer {
    flex: 0 0 auto; /* Don't grow or shrink */
}

/* Body section takes remaining height */
.hg-body {
    display: flex; /* Create the 3 columns */
    flex: 1 0 auto; /* Grow to fill space */
}

/* Fixed-width sidebars */
.hg-nav {
    flex: 0 0 200px; /* Don't grow or shrink, stay 200px */
    order: -1; /* Place before content */
}

.hg-sidebar {
    flex: 0 0 200px; /* Don't grow or shrink, stay 200px */
}

/* Fluid middle content */
.hg-content {
    flex: 1; /* Grow to fill available space */
}
/* Tablet Layout */
@media (max-width: 992px) {
    .hg-body {
        flex-direction: column; /* Stack vertically */
    }
    
    .hg-nav, .hg-sidebar {
        flex: 0 0 auto; /* Auto height */
        width: 100%; /* Full width */
    }
    
    /* Optional: Reorder for mobile */
    .hg-nav {
        order: 0; /* Reset order */
    }
    
    .hg-sidebar {
        order: 2; /* Move to bottom */
    }
    
    .hg-content {
        order: 1; /* Keep in middle */
    }
}

/* Mobile Layout */
@media (max-width: 576px) {
    /* Further adjustments for very small screens */
    .hg-header, .hg-footer {
        padding: 1rem;
    }
}

‚úÖ Best Practices

Use min-height: 100vh

Ensures the layout fills the entire viewport height, even with minimal content.

Set flex-basis on sidebars

Use flex: 0 0 [width] to prevent sidebars from growing or shrinking.

Use flex: 1 for content

Allows the main content to take up all available space between sidebars.

Consider mobile-first

Start with a single column layout and add complexity for larger screens.
